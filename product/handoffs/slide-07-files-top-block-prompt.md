# Промпт на зміни — слайд 7, верхній блок «Файли: види і повторне використання»

## Контекст для AI

Ти редагуєш слайд деки із заголовком **«Файли: види і повторне використання»**. Змінюй **тільки верхній блок** (вступний абзац + картки одразу під ним). Решту сторінки — секції «Компонент проти темплейту» та «Успадкування на старті продукту» — **не чіпай**.

## Що зараз

Верхній блок містить:
- Вступний абзац: «Будь-який файл — рівноправний член продукту. Зовнішні посилання (Figma, Google Doc, Techpacker) приймаються нарівні із завантаженими. Дві речі створюються прямо в Midpack.»
- Дві картки: `НОТАТКА · .MD` і `ПРОСТА ТАБЛИЦЯ`.

## Проблема

Вступ згадує три походження «файлу» (завантажений / зовнішнє посилання / народжений у Midpack), але картки показують лише два типи, народжені в Midpack. Найпоширеніший випадок (будь-який завантажений файл) і зовнішні посилання заховані в тексті. **Версіонування** не згадане взагалі, хоча саме воно робить «файл — рівноправний член продукту» правдою.

## Що зробити

Переробити верхній блок на структуру **«три види файлу, одна полиця»**: один вступний рядок + **три картки** (замість двох). Це дає окреме місце версіонуванню (картки 1–2) і окремий блок для URL (картка 3). Ключова думка, що має з'явитися: *збережені файли версіонуються в Midpack; посилання — ні, бо першоджерело лежить деінде.*

### Розкладка

Три рівні колонки. Картка 2 містить два компактні підпункти (нотатка + таблиця). Eyebrow-лейбли — у тому ж стилі, що вже на сторінці (верхній регістр, моноширинний).

```
┌──────────────────────────────────────────────────────────────────────────┐
│  [вступний рядок]                                                          │
│  ┌─────────────────────┐  ┌─────────────────────┐  ┌─────────────────────┐ │
│  │ ЗАВАНТАЖЕНИЙ ФАЙЛ ·  │  │ СТВОРЕНО В MIDPACK   │  │ ЗОВНІШНЄ ПОСИЛАННЯ · │ │
│  │ БУДЬ-ЯКИЙ ТИП        │  │  • Нотатка · .md     │  │ URL                  │ │
│  │                     │  │  • Проста таблиця    │  │                     │ │
│  └─────────────────────┘  └─────────────────────┘  └─────────────────────┘ │
└──────────────────────────────────────────────────────────────────────────┘
```

### Точні тексти (використовуй дослівно)

**Вступний рядок:**
> Будь-який файл — рівноправний член продукту. У Midpack «файл» потрапляє в продукт трьома способами, і всі три стоять на одній полиці.

**Картка 1 — eyebrow `ЗАВАНТАЖЕНИЙ ФАЙЛ · БУДЬ-ЯКИЙ ТИП`:**
> Перетягни що завгодно — `.xlsx`, `.ai`, `.pdf`, `.png`, `.zip`. Формат не обмежений. Кожне нове завантаження лягає **новою версією** поверх старої: історія зберігається, видно, хто і коли оновив, можна відкотити до попередньої. Сам файл зберігається в Midpack.

**Картка 2 — eyebrow `СТВОРЕНО В MIDPACK`:**
> Не треба зовнішнього редактора — два легкі типи створюються прямо тут і **так само версіонуються**:
> - **Нотатка · `.md`** — простий документ для коротких рішень, інструкцій, нотаток із примірки.
> - **Проста таблиця** — рядки/стовпці без формул, коли потрібно щось структуроване, а повноцінний Excel — надлишок.

**Картка 3 — eyebrow `ЗОВНІШНЄ ПОСИЛАННЯ · URL`:**
> Дані лежать деінде — Figma, Google Doc, Techpacker. У продукт додається не копія, а **жива URL-адреса**. Midpack не зберігає й не версіонує сам файл: оригінал оновлюють там, де він живе, а посилання завжди веде на актуальну версію. Приймається нарівні із завантаженими.

## Обмеження

- Зберегти наявні тексти нотатки й таблиці (вони переїжджають у картку 2 без втрат).
- Не вигадувати нові факти про продукт поза цим промптом.
- Зберігати лаконічний голос деки з виділеними **жирним** ключовими словами.
- Версіонування заявляти **тільки** для карток 1 і 2. Для картки 3 явно сказати, що посилання НЕ версіонується в Midpack.
- Не торкатися секцій нижче верхнього блоку.

## Навіщо (для самоперевірки)

- Контраст «версіонується тут» (картки 1–2) проти «першоджерело деінде» (картка 3) — головна думка слайда; зараз вона невидима.
- URL отримує картку нарівні з рештою, що відповідає фразі «приймаються нарівні».
- Три картки вгорі дзеркалять три джерела успадкування внизу (ВІД КОЛЕКЦІЇ / ВІД ВОРКФЛОУ / ВІД ЕТАПІВ) — слайд читається через «трійки».
